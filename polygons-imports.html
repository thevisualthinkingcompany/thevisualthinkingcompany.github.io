2 Polygons Imports

In many cases, simply using read_sf() will work, but this function ensures we avoid many of the common issues with many polygon downloads. It’s useful for leaflet foremost, but standardizing the CRS and dimension count also avoids many basic geoprocessing errors which can be annoying to troubleshoot.

  • Read polygons with sf package (which automatically interprets file type), or interpret the WKT into geometry, depending on input. read_sf() will generally be used for processing outside geodata, and WKT will generally be used from queries to our database.
  • Remove 3rd z-dimension if present (often default exports create an empty or zero value for this column; leaflet can’t use polygons with it, even when empty)
  • Ensure we’re using “Web CRS
read_polygons <- function(data_file){
    if (is.character(data_file)) {
    sf <- read_sf(data_file)
  } else if (is.data.frame(data_file) && "Geometry_WKT" %in% colnames(data_file)) {
    sf <- st_as_sf(data_file, wkt = "Geometry_WKT")
  } else {
    stop("Input must be either a file path or a data.frame with a 'Geometry_WKT' column.")
  }
  flat <- st_zm(sf,drop=TRUE)
  if (st_crs(flat)$epsg != 4326) {
    flat <- st_transform(flat, crs = 4326)
  }
  return(flat)
}
